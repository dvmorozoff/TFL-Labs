Входные данные - имя текстового файла формата:
1-ая строка - алфавит без epsilon, без пробелов: ab | 0123456789 и т.п.
Последующие строки - ответы на запросы функций membership и equivalence, т.к. МАТ'а нет.

Прикреплён файл "ab.txt", представляющий собой тест для задачи с консультации, кол-во букв a кратно 2, кол-во букв b ровно одно.

Код выводит много информации, не стал комментировать строки для большей наглядности.

Во время выполнения кода формируется вывод вида:
префикс : суффикс : значение мембершип для суммы : номер строки из текстового документа (удобно было для отладки и создания самого теста)

В конце:
Словарь, представляющий собой таблицу классов эквивалентности, где ключи - префиксы из верхней части таблицы, 
нулевой элемент их списка - строка из таблицы эквивалентности. 
Последующие элементы списка - префиксы из нижней части таблицы с идентичной строкой в таблице к.э.

Таблица, которая будет передаваться МАТ'у по запросу эквивалентности в виде:
(префикс + суффикс) (значение мембершип)

И также суффиксы.
